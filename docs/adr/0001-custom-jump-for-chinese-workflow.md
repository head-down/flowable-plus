# 使用自定义跳转逻辑实现中式审批流转，而非依赖 BPMN 预设连线

在 BPMN 模型上预设驳回边界事件连线是 Flowable 社区的主流方案。但这种方案要求所有流程设计者在每个 userTask 上手动配置回退路径，且驳回目标在运行时固定不变——无法动态选择"上一审批节点"或"发起人节点"。

我们选择通过 BPMN 模型回溯 + Flowable 的 ChangeActivityStateBuilder 实现运行时动态节点跳转。驳回目标由引擎在运行时通过模型遍历和历史数据混合计算得出，对流程定义零侵入。

## 考虑过的替代方案

- **BPMN 边界事件连线**：规范化但无法支持动态驳回目标，要求每个流程设计者预先绘制所有回退路径，实际落地阻力极大。
- **纯历史记录查找**：执行效率高，但在排他网关和并行网关等复杂拓扑下可能找到错误的目标节点。
- **Flowable 内部 impl API**：灵活度最高但与 Flowable 版本深度耦合，引擎升级随时可能破坏。

## 后果

- 核心模块需要访问 RepositoryService（解析 BPMN 模型）和 HistoryService（补全网关分支判定），形成 BPMN 模型 + 历史记录的混合查找策略。
- 对 Flowable 的 ChangeActivityStateBuilder 有版本依赖（6.4+），本项目基准版本 6.8.0 满足此要求。
- 未来支持子流程、调用活动等复杂拓扑时需要扩展 NodeFinder 的遍历能力。
