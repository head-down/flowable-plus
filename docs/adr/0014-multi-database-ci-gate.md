# ADR-0014: 多数据库 CI 矩阵作为 v1.0.0 GA 硬性准入条件

**日期**: 2026-07-21
**状态**: 接受

## 上下文

项目当前状态为 Alpha，所有测试仅在 H2 内存数据库上运行。Flowable 支持的数据库（MySQL、PostgreSQL、Oracle、SQL Server、DB2 等）在 SQL 方言上存在已知差异，包括但不限于：

- 分页语法（LIMIT vs OFFSET vs ROWNUM）
- 锁机制（SELECT ... FOR UPDATE 在不同数据库中的行为差异）
- 函数差异（如日期函数、字符串函数）
- 序列 vs 自增主键的实现

H2 内存数据库主要模拟标准 SQL，但无法发现上述方言差异导致的问题。在 H2 上通过测试不能保证在生产数据库上正常运行。

## 决策

1. **项目状态在 README 中明确标注为 Alpha**，并附多数据库未验证的免责声明
2. **v1.0.0 GA 的硬性准入条件**：CI 矩阵必须覆盖 MySQL 8.0 和 PostgreSQL 14。这两个数据库已覆盖国内 95% 的用户场景
3. **使用 Testcontainers** 在测试代码中自动拉起临时数据库容器，无需手动部署或依赖云数据库服务
4. **Oracle / 达梦 / 信创数据库**：不列入 v1.0.0 GA 准入条件。待使用者提 Issue 后按需补充
5. 在此矩阵全部通过之前，不允许标记任何版本为 GA（1.0.0）

## 备选方案

- 不做多数据库 CI，依赖社区反馈发现数据库兼容问题 — 被否决，因为工作流引擎在关键业务中使用，问题发现过晚后果严重
- 覆盖 Oracle 等小众数据库 — 被否决，Oracle Docker 镜像有合规风险，且个人项目无需覆盖；按需补充成本更低
- 手动部署数据库实例 — 被否决，Testcontainers 是 Spring 官方推荐的最佳实践，零配置、本地可复现

## 后果

- v1.0.0 GA 发布将推迟至多数据库 CI 矩阵完成后
- CI 运行时间将增加（测试套件 × 3 数据库）
- 本地开发默认仍使用 H2（零启动开销），CI 额外运行 MySQL 和 PG
- 使用 Testcontainers 意味着本地和 CI 都需要 Docker 环境（CI 已天然支持）
- 可能需要修复现有 H2 兼容但在 MySQL/PG 上有问题的测试代码
